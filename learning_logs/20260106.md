# 상속(inheriment)
- 객체지향 프로그래밍의 핵심개념중 하나로, 기존 클래스의 필드의 메소드를 재사용하여 새로운 클래스를 만드는 것을 의미함

## 상속의 필요성
- 상속을 통해 공통적인 기능을 부모 클래스에 정의하고, 이를 '여러' 자식 클래스가 재사용가능
    -   동물 클래스가 있고 이를 상속받는 개, 고양이 클래스가 있다고 가정한다면, 동물의 공통적인 field(이름/나이 등)와 행위를 부모 클래스에 정의 하고, 개와 고양이는 각각 고유한 특성을 추가할 수 있다.
- 이상의 과정을 거쳤을때 가질 수 있는 이점 :
    - 중복된 코드를 줄일 수 있다.


## 특징
1. 단일 상속
        - java는 단일 상속만 지원함. 하나의 클래스는 하나의 부모 클래스만 상속 받을 수 있어 상속 구조가 간단하고 명확함
2. super 키워드
        - 클래스의 생성자 및 setter에서 학습한 this와 유사한 개념으로 부모 클래스의 field에 접근하거나 부모 클래스의 생성자를 호출하거나/
3. _method override_
        - 자식 클래스에서 부모 클래스의 기본 동작을 재정의할 수 있다.
        이를통해 자식 클래스는 부모 클래스의 기본동작을 자신만의 방식으로 변경가능
        - 예시
            - Animal 클래스의 걷기 method를 Dog가 상속 받고, 네발로 걷기 등으로 재정의
4. final 키워드 : 클래스에 final이 붙으면 상속불가
5. method에 final이 붙는 경우에는 override 불가능

## 장점
1. 코드의 재사용성 
    -  동일한 코드를 기본적으로 작성할 필요
2. 유지 보수의 용이성
    - 부모클래스의 변경 사항이 자식클래스에 자동으로 반영된다.
3. 객체 간 관계 표현 
    - IS-A관계를 명확히 나타냄
    - IS-A : 개는 동물의 일종이다. 와 같은 예시로 IS-A는 is a kind of의 축약어에 해당
    - 그래서 우리가 미리 만들어둔 클래스를 예를 들었을때, human과 tiger는 animal의 자식클래스라 볼 수 있다.
