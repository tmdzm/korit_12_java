# Java Generic Wildcard

## 제네릭

```java
public class Box<T> {
//field 선언
    private T value;
    public void setValue(T value){
        this.value = value;
    }
    public T getValue(){
        return value;
    }
}
```
과 같은 방식으로 작성했을 때 value의 자료형을 고정시키지 않고 사용 가능함. 예를 들어 value에 string / Integer / Double/ Product/ Item...등으로 사용가능.<br>
즉 Java 자체의 자료형 뿐만 아니라 개발자가 정의한 것도 집어넣을 수 있습니다.<br>
이는 추후 백엔드 설계에도 매우 중요합니다.

## 와일드 카드
- 이상의 제네릭 타입을 사용할 때 구체적인 타입을 명시하지 않고 범용적으로 처리하고 싶을 때 사용함.<br>
'?' 사용

```java
public static void main(String[] args){
    List<?> exampleList = new ArrayList<String>();
}

```
### 종류
1. 모든 타입 허용
    - 읽기 전용. 어떤 타입이든지 일단 받아놓고 분류해야 할 때
2. T또는 T의 서브타입(자식 클래스)
    - 읽기 전용. 제한된 타입에서 읽을 때
3. T또는 T의 슈퍼타입(부모 클래스)
    - 쓰기 전용. 제한된 타입으로 넣을 때

# Collections
- 여러 객체들을 모아둔 것들을 의미함 -> 배열과는 약간 차이가 있다.
- 다만 java에서의 List는 python에서의 List와도 차이가 있다.

```java
List<Person> list1 = new ArrayList<>();
```

## 종류
1. List : 순서가 있는 데이터의 집합으로, 중복된 요소를 허락함
    - ArrayList : 배열기반 리스트
    - LinkedList : 연결리스트
2. Set  : 중복을 허용하지 않는 데이터들의 집합
    - HashSet : 해시 테이블 기반의 집합. element의 순서를 보장하지 않음
    - TreeSet : 이진 검색 트리기반의 집합, element가 정렬된 상태로 유지
3. Map : key

### 참조 : JavaScript에서의 객체 정의방식
```js
const person1 = {
    name : "김일",
    age  : 20
};
```
- Java에서의 Map과 유사한 형태를 가진다.
- 근데 JavaScript에서의 key-value쌍을 property라고 부른다.
- 근데 Java에서의 Map의 key-value쌍을 entry라고 부른다.
- 그러면 Java는 객체에서 field와 field의 value라고 볼 수 있다.
- 근데 또 python에서는 attribute의 value입니다.
